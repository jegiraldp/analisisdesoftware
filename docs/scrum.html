Roles Historias de Usuario User Story Mapping Eventos Artefactos

1. Roles en el Scrum Team

Product Owner (PO)

Maximiza el valor del producto y gestiona el Product Backlog.

Scrum Master

Líder que sirve al equipo y elimina impedimentos.

Team

Personas comprometidas a crear el Incremento útil cada Sprint.

2. Historias de Usuario (HU)

Breve descripción de una funcionalidad contada desde la perspectiva del usuario.

Buenas Prácticas (Modelo INVEST)

  • Independiente, Negociable, Valiosa, Estimable, Small (Pequeña), Testeable.
  • Utilizar la estructura: "Como [rol], quiero [acción], para [beneficio]".
Ejemplos:
  1. Como Cliente, quiero filtrar productos por precio, para encontrar lo que se ajuste a mi presupuesto.
  2. Como Administrador, quiero suspender cuentas de usuario, para mantener la seguridad de la plataforma.
  3. Como Vendedor, quiero ver un historial de mis ventas, para llevar control de mis comisiones.

Plantilla de Documentación de HU

HU-05: Filtrado de Productos
Como: Cliente Quiero: Filtrar por categorías
Para: No perder tiempo viendo productos que no me interesan.
Criterios de Aceptación:
- El sistema muestra un menú lateral con las categorías disponibles.
- Al seleccionar una categoría, la lista de productos se actualiza instantáneamente.

3. User Story Mapping

Técnica visual que toma como insumo los Requisitos Funcionales y No Funcionales.

1. Épica Nivel Estratégico / Gran funcionalidad
2. Historia de Usuario Nivel de Valor / Entregable del Sprint
3. Actividad / Tarea Nivel Operativo / Desglose técnico

4. Eventos (Ceremonias)

Son bloques de tiempo (time-boxed) diseñados para crear regularidad y minimizar reuniones no definidas.

Documentación en Tabla de Eventos

Evento Duración Máx. Propósito Principal
Sprint 1 mes o menos Contenedor de todos los demás eventos. Creación del Incremento.
Sprint Planning 8h (Sprint 1 mes) ¿Qué se hará y cómo se hará en este Sprint?
Daily Scrum 15 min Inspeccionar el progreso hacia el Sprint Goal y ajustar el plan diario.
Sprint Review 4h (Sprint 1 mes) Presentar resultados a los interesados y colaborar en qué hacer después.

5. Artefactos y sus Compromisos

Representan trabajo o valor para proporcionar transparencia.

1. Product Backlog: Lista ordenada de todo lo que se conoce que es necesario en el producto. (Compromiso: Product Goal).

2. Sprint Backlog: Conjunto de elementos del Product Backlog seleccionados para el Sprint. (Compromiso: Sprint Goal).

3. Incremento: La suma de todos los elementos del Product Backlog completados durante el Sprint. (Compromiso: Definition of Done).

Ejemplos de Aplicación

  • Product Goal: "Llegar a 10,000 usuarios activos en la app móvil para fin de año".
  • Sprint Goal: "Habilitar el proceso de registro mediante redes sociales".
  • Definition of Done (DoD): "Código revisado, pruebas unitarias al 100% y documentación técnica actualizada".
